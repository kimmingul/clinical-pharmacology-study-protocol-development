### [Major-1] 절단 AUC₀₋₇₂ₕ의 CV% 사용 — AUC₀₋ₜ CV와의 구분 미명확

- **심각도**: Major
- **위치**: 섹션 13.1.2 (산출 파라미터), 통계 설계 문서 섹션 2.2
- **내용**:  
  Sample size 계산에서 AUC의 intra-subject CV로 "AUC₀₋ₜ CV 7.5–11.6% → 12% 사용"이라고 기재되어 있다. 그러나 본 계획서의 1차 평가변수는 **AUC₀₋ₜ가 아닌 AUC₀₋₇₂ₕ (절단 AUC)**이다. 절단 AUC의 CV가 전체 AUC₀₋ₜ의 CV와 다를 수 있음에도 불구하고, 이 차이에 대한 명시적 논의 없이 AUC₀₋ₜ의 CV 값을 그대로 활용하였다.  
  
  biostatistician agent 정의 문서는 다음과 같이 명시하고 있다: "절단 AUC의 CV가 전체 AUC와 다를 수 있으므로, 가능하면 AUC₀₋₇₂ₕ의 CV 사용." 계획서 어디에도 AUC₀₋₇₂ₕ에 특정한 CV 값을 확인하거나 그 대용값 사용의 합리적 이유를 기술하지 않았다.  
  
  실질적으로 Sample size는 Cmax (CV=18%)를 기준으로 결정되어 있어, AUC의 CV 차이가 최종 대상자 수(36명)에는 영향을 주지 않는다. 그러나 **1차 평가변수에 사용된 CV의 출처를 명확히 기술하지 않으면 규제 심사 시 방어 논거가 약해진다**.

- **근거**:  
  - Biostatistician agent 정의 (`.claude/agents/biostatistician.md`): "절단 AUC의 CV가 전체 AUC와 다를 수 있으므로, 가능하면 AUC₀₋₇₂ₕ의 CV 사용"  
  - FDA Guidance on BE (2014): truncated AUC에 대한 CV는 절단 시점 기준의 문헌값 또는 적절한 추정치를 사용할 것을 권고  
  - Chow & Liu, 2009 Chapter 5: CV는 1차 평가변수 기준으로 결정

- **권고**:  
  계획서 섹션 13.1.2에 다음 내용을 추가하라:  
  1. AUC₀₋₇₂ₕ는 0–72시간 구간으로 절단된 AUC이므로, 해당 구간의 intra-subject CV는 AUC₀₋ₜ의 CV와 실질적으로 유사하거나 더 낮을 것으로 예상됨을 문헌 근거와 함께 기술.  
  2. 문헌에서 AUC₀₋₇₂ₕ 전용 CV가 보고되지 않은 경우, AUC₀₋ₜ CV(12%)를 보수적 대용값으로 사용한다는 명시적 논거 추가.  
  3. Sample size가 Cmax (CV=18%) 기준으로 결정되었고, AUC₀₋₇₂ₕ용 36명이 충분함을 별도 표로 제시.

### [Major-2] Sample size 계산 공식의 정밀도 — ⌈n_per_seq / (1−dropout)⌉ 적용 방식

- **심각도**: Major
- **위치**: 섹션 13.1.3 (산출 결과)
- **내용**:  
  계획서는 다음 보정 공식을 사용한다:

  ```
  n_total_adjusted = ⌈n_total / (1 − dropout_rate)⌉
  ```

  계산 결과를 보면: n per sequence(순수) = 14명, 보정 후 18명. 즉 ⌈14 / 0.80⌉ = ⌈17.5⌉ = 18명으로 계산하였다.  
  
  그런데 계획서 섹션 13.1.3 표에는 "⌈14 / 0.80⌉ = 18"로 표기되어 있으나, 통계 설계 문서에는 "n total (순수) 28명 → 탈락률 20% 보정: ⌈28 / 0.80⌉ = 35 → 짝수 조정 36명"이라고 기술되어 있다. 두 문서 간에 보정 계산 단위가 다르다:

  - 계획서: sequence당 14명을 보정 → 18명 × 2 = **36명** (올바름)  
  - 통계 설계 문서: 전체 28명 보정 → ⌈28/0.80⌉ = ⌈35⌉ = 35명 → **짝수 조정으로 36명** (올바름이나 과정 설명 불일치)  

  두 경로 모두 최종 36명으로 일치하나, "35 → 짝수 조정 36명"이라는 설명이 계획서 본문에는 없어 계산 추적이 어렵다. 또한 "짝수 조정(even number adjustment)" 근거(2×2 crossover에서 두 sequence를 동일 크기로 유지해야 함)가 계획서에 명시되어 있지 않다.

- **근거**:  
  - 2×2 crossover 설계에서 sequence별 대상자 수는 1:1 배정 유지를 위해 짝수이어야 함 (Chow & Liu, 2009)  
  - 계산 과정의 완전한 기록은 규제 제출 시 필수 (MFDS 생동성 기준)

- **권고**:  
  계획서 섹션 13.1.3에 다음을 명확히 기술하라:  
  - "⌈28 / 0.80⌉ = 35명이나, 2-sequence 균형 배정(1:1)을 위해 짝수로 올림 → **36명** (TR 18명, RT 18명)"  
  - 또는 "n per sequence = ⌈14 / 0.80⌉ = 18명 → 총 36명"의 단계별 계산 명시

### [Minor-1] ANOVA 모형에서 Subject(Sequence) 효과의 명명 불일치

- **심각도**: Minor
- **위치**: 섹션 13.3.2 (분석 모형)
- **내용**:  
  계획서 수식에는 `SUBJ(SEQ)_ij`로, 효과 설명 표에는 `SUBJ(SEQ)_ij | 대상자 효과 (sequence 내 내포) | 변량`으로 기술되어 있다. 이는 통계적으로 올바른 표현이다. 그러나 통계 설계 문서(섹션 5.2)의 수식에서는 `Subject(Sequence)` 표현이 모형 수식 밖 표에 분리되어 기술되어 있어, 수식 표현과 설명 표 사이의 연결이 명확하지 않다.  
  
  SAS PROC MIXED 코드에서는 `RANDOM SUBJECT(SEQ) / SUBJECT=SUBJECT TYPE=VC;` 또는 이와 동등한 구문으로 구현되어야 한다. 계획서에 소프트웨어 구문 수준의 명세는 없어도 무방하나, 수식과 효과 목록이 완전히 정렬되어야 한다.

- **근거**:  
  - 2×2 crossover BE ANOVA 표준 모형: SEQ, SUBJ(SEQ: random), PER, FORM (Patterson & Jones, 2006; FDA Draft Guidance)  
  - 모형 수식의 모든 항이 효과 목록과 1:1 대응되어야 함

- **권고**:  
  계획서 섹션 13.3.2의 수식을 다음과 같이 완전한 형태로 정렬하라:

  ```
  ln(Y_ijk) = μ + SEQ_i + SUBJ(SEQ)_ij + PER_j + FORM_k + ε_ijk
  ```

  효과 설명 표에 `SUBJ(SEQ)_ij` 행을 수식과 동일한 위치(순서)에 명시하고, "(변량 효과 — SAS PROC MIXED에서 RANDOM 문으로 지정)"임을 주석으로 추가.

### [Minor-2] Tmax 분석 방법 — Wilcoxon Signed-Rank Test의 검정 방향 불명확

- **심각도**: Minor
- **위치**: 섹션 13.4 (2차 분석)
- **내용**:  
  "Tmax: Wilcoxon Signed-Rank Test (비모수 검정), 목적: 중앙값 비교"로 기술되어 있다. Tmax는 로그 변환이 적절하지 않고 분포가 비정규적이어서 비모수 검정 선택은 적절하다. 그러나 다음 사항이 미명확하다:

  1. **단측 vs 양측**: 동등성 목적(equivalence)인지, 단순 차이 검정(difference)인지 명시되지 않음  
  2. **보고 지표**: 중앙값 차이(median difference) 및 95% CI(또는 90% CI) 보고 여부  
  3. **Tmax의 동등성 기준**: Tmax는 2차 평가변수로 동등성 기준이 없어 참고 목적임을 명시해야 함

- **근거**:  
  - EMA BE Guideline (2010): Tmax는 참고 파라미터로, 중간값(median) 및 범위(range) 기술통계를 제시하며 비모수 검정을 수행함  
  - 동등성 가설(equivalence hypothesis)이 아닌 차이 검정(difference test) 수행 시 해석 주의 필요

- **권고**:  
  섹션 13.4 Tmax 행을 다음과 같이 보완:  
  - 분석 방법: "Wilcoxon Signed-Rank Test (양측, α=0.05); 중앙값 차이 및 95% CI 보고"  
  - 목적 설명에 "동등성 판정 목적이 아닌 참고 비교"임을 명시

### [Minor-3] 결측치 처리 — "Cmax 시점 누락" 처리 기준의 모호성

- **심각도**: Minor
- **위치**: 섹션 13.5.1 (결측치 분류 및 처리 원칙)
- **내용**:  
  "Cmax 시점 채혈 실패: 해당 period PK 분석 제외, PP Set 제외 고려"에서 **"고려"**라는 표현이 모호하다. PP Set 제외 기준은 사전에 명확히 정의되어야 한다. 또한 "Cmax 시점"이 단일 시점(예: 8h)인지 Tmax 구간(예: 6–12h 중 3개 이상 누락)인지가 불명확하다.  
  
  본 계획서의 PK 채혈 설계는 Tmax 구간(6–12h)에 7개 시점을 배치하여 Cmax를 추정하므로, 단 1–2개 시점 누락 시에도 Cmax를 실측값으로 확인할 수 있는 가능성이 있다. 따라서 단일 시점 누락 ≠ Cmax 누락임을 명확히 구분해야 한다.

- **근거**:  
  - MFDS 생동성 기준: 프로토콜 위반(protocol deviation)과 PP Set 제외 기준은 사전에 SAP에 정의  
  - ICH E9: 결측치 처리 원칙은 사전 명시 원칙(pre-specification) 준수 필요

- **권고**:  
  섹션 13.5.1에 다음을 추가하라:
  1. "Cmax 시점"을 "Tmax 예상 구간(6–12h) 내 시점"으로 정의하고, 해당 구간에서 몇 개 시점이 누락될 경우 PP Set에서 제외하는지 기준을 명시  
  2. "PP Set 제외 고려" → "프로토콜에서 사전 정의한 중대한 프로토콜 위반 기준에 해당 시 PP Set 제외"로 변경  
  3. 상세 PP Set 제외 기준 목록을 SAP에 명시한다는 참조 문구 추가

### [Minor-4] 캐리오버 효과 검정의 유의수준 선택 — α=0.10 근거 미기재

- **심각도**: Minor
- **위치**: 섹션 13.6 (캐리오버 효과 평가)
- **내용**:  
  "Sequence × Period 교호작용 검정 (α=0.10 기준)"로 기술되어 있다. α=0.10을 사용하는 것은 관행적으로 허용되나, 그 이유가 명시되지 않았다. α=0.10을 사용하는 근거는 캐리오버 검정의 낮은 검정력(통상 underpowered)을 보완하기 위한 것이나, 이를 계획서에서 설명하지 않으면 심사자의 질의를 받을 수 있다.  
  
  또한, "유의한 carry-over 의심 시 1st period 데이터만으로 추가 분석 실시 (참고용)"는 현재 통계학 문헌에서 권고되지 않는 방법이다. Senn (2002)은 carry-over 검정 결과에 따른 조건부 분석(conditional analysis) 전략이 Type I error를 통제하지 못할 수 있다고 지적한다. 이를 "참고용"으로만 사용한다면 그 한계를 명시해야 한다.

- **근거**:  
  - Senn S. *Cross-over Trials in Clinical Research*, 2nd ed. Wiley, 2002: 캐리오버 검정의 낮은 검정력 및 조건부 분석의 한계  
  - 21일 washout이 충분히 설정된 경우(10×t½ 충족), 통계적 carry-over 검정보다 설계 기반의 carry-over 배제가 더 강력한 근거임

- **권고**:  
  섹션 13.6에 다음을 추가하라:
  1. "α=0.10 선택 근거: 2×2 crossover에서 교호작용 검정은 검정력이 낮으므로, 보다 관대한 유의수준(α=0.10)을 사용하여 carry-over의 신호를 민감하게 감지한다"  
  2. "1st period 데이터만의 분석은 통계적 타당성이 제한적이므로(Senn, 2002), 결과 해석은 탐색적 참고 수준에 그침을 명시"  
  3. "본 시험은 21일 washout(10×t½ 충족)으로 carry-over 효과를 설계 수준에서 충분히 배제하였음"을 맥락으로 기술

### [Minor-5] 분석 집단 정의 — PP Set에서 "중대한 프로토콜 위반" 기준 미명세

- **심각도**: Minor
- **위치**: 섹션 13.2 (분석 집단)
- **내용**:  
  PP Set 정의: "PK Set 중 중대한 프로토콜 위반이 없는 대상자"로 기술되어 있다. 그러나 "중대한 프로토콜 위반"의 구체적 기준이 계획서 본문 어디에도 열거되어 있지 않다. MFDS 및 ICH E9는 PP Set 정의 시 제외 기준을 사전에 명세할 것을 요구한다.  
  
  예상 프로토콜 위반 예시(빠진 항목들): 병용 금지 약물 복용, 구토 발생, 채혈 시점 대규모 이탈, 투여 불순응(non-compliance) 등.

- **근거**:  
  - ICH E9 (R1): Analysis Population 정의는 구체적 제외 기준을 사전 명시 필요  
  - MFDS 생동성 기준: PP Set 제외 기준을 SAP에 사전 확정 요구

- **권고**:  
  섹션 13.2에 PP Set 제외 기준의 예시 목록을 추가하거나, "상세 PP Set 제외 기준은 SAP(Statistical Analysis Plan)에 데이터베이스 잠금 전 사전 확정한다"는 참조 문구를 명시하라.

### [Minor-6] 통계 소프트웨어 대안 제시 — SAS/R 병기 시 일관성 보장 절차 미기술

- **심각도**: Minor
- **위치**: 섹션 13.8 (통계 소프트웨어)
- **내용**:  
  "SAS® 9.4 PROC MIXED 또는 R `lme4`/`nlme`"와 같이 복수의 소프트웨어를 대안으로 제시하였다. 다만 어떤 소프트웨어를 1차로 사용할지, 두 소프트웨어 결과 간 불일치 시 처리 절차가 명시되어 있지 않다. 규제 제출 관점에서는 사전에 1차 소프트웨어를 확정하는 것이 바람직하다.

- **근거**:  
  - MFDS 생동성 기준: 통계분석에 사용된 소프트웨어 및 버전을 최종 보고서에 명시 요구  
  - 실무 관행: 복수 소프트웨어 사용 시 결과 일치 여부 교차 검증 절차 명시 권고

- **권고**:  
  소프트웨어 표에 "1차 분석 소프트웨어"를 명시하고, "2차 소프트웨어는 검증 목적으로 사용하며, 결과 불일치 시 SAP에 정의된 기준을 따른다"는 문구를 추가.

## 종합 의견

### 전반적 평가

Amlodipine BE 계획서의 통계 섹션은 전반적으로 **잘 구조화되어 있으며**, 표준 2×2 crossover BE 시험의 통계 방법론을 충실히 따르고 있다. 주요 항목별 평가:

| 항목 | 평가 | 비고 |
|------|------|------|
| Sample size 산출 공식 | 적절 | Chow & Liu (2009) 공식 명시, 코드 추적 가능 |

### Sample size 독립 검증

계획서 제시값을 공식으로 직접 검증한 결과:

```
sigma_w = sqrt(ln(1 + 0.18^2)) = sqrt(ln(1.0324)) = sqrt(0.03188) = 0.17856
delta = |ln(0.95)| = 0.05129
theta = ln(1.25) = 0.22314
theta - delta = 0.17185

n_per_seq = ceil((1.6449 + 0.8416)^2 * 2 * 0.17856^2 / 0.17185^2)
          = ceil(6.1853 * 2 * 0.031883 / 0.029532)
          = ceil(6.1853 * 0.063766 / 0.029532)
          = ceil(6.1853 * 2.1590)  ← 오류 가능. 재계산:
          = ceil((2.4865)^2 * 2 * 0.031883 / 0.029532)
          = ceil(6.1827 * 0.063766 / 0.029532)
          
실제 계산:
  (z_alpha + z_beta)^2 = (1.6449 + 0.8416)^2 = (2.4865)^2 = 6.1827
  분자: 6.1827 * 2 * 0.031883 = 6.1827 * 0.063766 = 0.39420
  분모: 0.17185^2 = 0.029532
  n_per_seq = ceil(0.39420 / 0.029532) = ceil(13.35) = 14명 ✓
```

**계획서 제시 n=14명/sequence (순수), 총 28명 (순수), 36명 (보정)은 계산상 정확하게 재현됨.** 수치 오류 없음.
